## 1. 研究目标

**核心问题**：在过去约 10 年的中国市场里，**哪些公募基金经理**持续创造了 **风险调整后的超额收益（alpha）**？他们采用了什么 **可解释、可复制的策略**？所在的 **基金公司** 在投研体系上有什么特征？

**三个产出层次**：

1. **L1 - 候选池**：风险调整收益 Top 30-50 的在管基金经理名单
2. **L2 - 深度画像**：每位候选的策略归因（行业、风格、因子暴露）、代表产品历年表现、熊市/牛市分阶段表现
3. **L3 - 基金公司洞察**：候选集中的"明星厂牌"（兴全、易方达、汇添富、中欧、睿远、广发……）的投研文化与可复制性

## 2. 范围与边界

### 2.1 包含

| 维度 | 标准 |
|---|---|
| 基金类型 | 偏股混合型、普通股票型、灵活配置型、增强指数型 |
| 任职起点 | 至少在 **2016-01-01** 之前开始管理本基金（确保 10 年完整周期） |
| 现任状态 | **仍在管**（截至当前季度），已离任者不计入最终名单（可在"历史参考"栏备注） |
| 投资范围 | A 股为主，可包含港股通；不包含 QDII 境外产品 |
| 规模 | 暂不设下限，但 ≤2 亿的小微基金需在备注中标注"流动性风险" |

### 2.2 排除

- 纯债、货币、被动 ETF（除非是增强指数）
- 任职不足 10 年的基金经理（即便业绩亮眼）
- 私募、券商资管、信托产品
- 量化对冲类产品（策略异质性过强，本研究聚焦主观多头）

### 2.3 时间窗口的关键节点（用于子区间测试）

| 区间 | 风格 | 用途 |
|---|---|---|
| 2016-01 ~ 2018-01 | 价值/蓝筹修复 | 周期一致性 |
| 2018-01 ~ 2019-01 | 熊市（贸易战） | 下行风险测试 |
| 2019-01 ~ 2021-02 | 核心资产牛市 | 牛市 alpha 区分 |
| 2021-02 ~ 2022-10 | 成长→价值切换 | 风格切换适应性 |
| 2022-10 ~ 2024-09 | 防御/红利/微盘 | 新兴风格识别 |
| 2024-09 ~ 2026-06 | 政策驱动反弹 | 最新风格延续性 |

## 3. 评价框架

### 3.1 主指标（必须计算）

| 指标 | 公式要点 | 阈值参考 |
|---|---|---|
| **年化收益** | 几何年化 | Top 10% 分位 |
| **年化波动率** | 净值日收益标准差 × √252 | 越低越好（同收益下） |
| **最大回撤** | 区间内峰谷最大跌幅 | Top 10% 分位 |
| **Sharpe** | (年化收益 - 3%) / 年化波动率 | **≥ 1.0 为优秀，≥ 1.5 为顶级** |
| **Calmar** | 年化收益 / \|最大回撤\| | **≥ 1.0 为优秀，≥ 2.0 为顶级** |
| **信息比率 (IR)** | (基金收益 - 基准) / 跟踪误差 | **≥ 1.0 为优秀** |
| **胜率** | 月度跑赢基准比例 | 参考指标 |

### 3.2 基准选择

- **偏股混合 / 普通股票** → 沪深 300（000300）+ 中证偏股基金指数（930950）双基准
- **灵活配置** → 沪深 300 + 50/50 沪深 300/中证 500
- **增强指数** → 跟踪指数本身

### 3.3 为什么"风险调整后收益"优先于"绝对收益"？

- 避免"高仓位赌赛道"的伪 alpha（如 2019-2020 押注白酒/新能源的幸运儿 vs 真正持续 alpha）
- 剔除 beta 暴露后，剩余收益更接近"可复制的技能"
- 熊市/震荡市的表现是 alpha 的真正试金石

## 4. 研究方法（5 个阶段）

### 阶段 1：数据源勘察与候选池构建

**目标**：拿到一份可信、可量化的"初筛名单"。

> ⚠️ **本阶段所有数据采集动作前，先查 [`data/SEARCH-GUIDE.md`](./data/SEARCH-GUIDE.md)**。该 SOP 包含：晨星页面提取模板、限速、避坑、JSONP 兜底。

**数据源（单源原则）**：

| 来源 | 用途 | 访问方式 |
|---|---|---|
| **晨星中国 `/fund/<6位代码>.html`** | **唯一数据源**——基金元信息、晨星评级（3/5/10年）、风险调整后指标（Sharpe/Calmar/Sortino/Alpha/Beta/IR）、业绩归因、重仓股、行业配置、经理、内部持有 | `chrome-devtools` navigate + evaluate_script |
| 公开访谈（雪球/聪明投资者/点拾投资/远川） | **辅助**：策略归因的定性证据 | `mcp__MiniMax__web_search` + 抓取 |

> **为什么单源**：
> - 避免不同平台数据差异（口径/起算日/调整方式）造成的"伪矛盾"
> - 单一可信基准（晨星评级）= 行业共识，避免争议
> - 一只基金 = 一个 URL = 一页数据 = 50+ 字段，研究效率最大化
>
> **不使用的源**（已 deprecated，保留在 SEARCH-GUIDE 附录作历史参考）：
> - ❌ 天天基金（fund.eastmoney.com）—— 改用晨星后无需
> - ❌ 巨潮资讯（cninfo.com.cn）—— 晨星页面已含持仓/经理，无需再查季报
> - ❌ 好买/同花顺/私募排排网 —— 第三方数据可能与晨星冲突
> - ❌ 基金公司官网 —— 营销文案，参考价值低

**初筛标准**：
1. 偏股/股票/混合型基金（晨星分类非"货币/纯债/被动"）
2. 任职起始日 ≤ 2016-01-01
3. 基金成立日 ≤ 2016-06-19（让 10 年晨星评级可得；老基金更佳）
4. 晨星页面能正常打开（用 SEARCH-GUIDE 场景 A 模板验证）

**输出**：初版候选池（预计 50-100 人），写入 `output/INDEX.md`。

---

### 阶段 2：定量打分与排序

**目标**：用风险调整后指标排出 Top 30-50。

**步骤**：
1. 对每只候选基金，用 SEARCH-GUIDE 场景 A 模板抓取晨星 `/fund/<code>.html` 全部字段
2. 落盘到 `data/raw/morningstar/<code>.json`（**立即落盘，不在内存累积**）
3. 按以下指标排序：
   - **主榜**：晨星 10 年评级（4-5 星） / 5 年评级
   - **辅榜 1**：夏普比率（同口径对比）
   - **辅榜 2**：卡玛比率（年化收益 / |最大回撤|）
   - **辅榜 3**：信息比率（Alpha 稳定性）
4. 多榜取交集 + 各榜独有
5. 跨产品聚合：经理名下多只产品取**规模最大**或**评级最高**一只为代表

**输出**：`output/rankings-top30.md` + `data/raw-metrics.csv`

---

### 阶段 3：风格与策略归因

**目标**：解释"为什么他能赚钱"，并验证可复制性。

> **关键简化**：晨星页面**已经包含大量归因数据**（行业归因、个股归因、风格箱、换手率、行业偏好、地区偏好）——直接读，不用自己算。

**归因维度**（晨星 `/fund/<code>.html` 已提供）：

| 维度 | 晨星字段 | 输出 |
|---|---|---|
| **业绩归因** | 行业配置 vs 个股选择（百分比） | 选股 alpha、择时贡献 |
| **持仓风格** | 股票投资风格箱（大盘/中盘/小盘 × 价值/平衡/成长） | 风格标签 |
| **换手率** | 基金换手率 % | 持仓周期（交易型 vs 持有型） |
| **行业偏好** | 晨星行业占比（前 10 大行业） | 行业标签 |
| **规模暴露** | 股票地区分布（中国大盘/中盘/小盘/海外） | 是否押注小盘 |
| **重仓股稳定性** | 重仓季度数（前 10 大重仓持有几个季度） | 持仓稳定性 |

**策略标签体系**（参考用）：
- 价值红利 / 质量价值 / GARP / 深度价值
- 成长（消费成长 / 科技成长 / 医药成长 / 新能源）
- 行业主题（消费 / 医药 / 科技 / 制造）
- 均衡配置 / 全行业选股型
- 小盘/微盘

**输出**：每位候选的"策略画像"附录，存入 `output/profiles/<经理>.md`

---

### 阶段 4：持续性测试与稳健性

**目标**：验证 alpha 是否穿越周期、是否依赖单一风格。

**测试**：
1. **滚动窗口**：1Y / 3Y / 5Y 滚动 Sharpe，看是否持续在 Top 50%
2. **子区间表现**：对照 §2.3 的 6 个子区间，识别"风格漂移"和"周期幸运儿"
3. **熊市表现**：2018、2022、2024 三轮下跌中相对最大回撤与反弹弹性
4. **风格切换测试**：2021-02 成长→价值切换、2024 微盘崛起两个时点的应对

**稳健性判据**：
- **顶级**：5Y 滚动 Sharpe 始终在 Top 20%，且 2 轮以上熊市跑赢基准
- **优秀**：3Y 滚动 Sharpe 持续 Top 30%，1 轮熊市显著跑赢
- **候选**：风格明确但在某区间表现一般

**输出**：`output/persistence-test.md`

---

### 阶段 5：基金公司层洞察

**目标**：识别"明星厂牌"，理解"为什么是他们"。

**分析**：
- 候选基金经理的所属基金公司分布（哪家出"长跑选手"最多？）
- 公司投研体系特征：投决会、基金经理独立决策权、研究员覆盖深度、激励机制
- 是否有"师承关系"（如兴全系、汇添富系）
- 公司层面的产品线布局：是否给经理足够长的时间证明自己

**输出**：`output/firm-insights.md`

---

### 阶段 6（可选）：交叉验证与终稿

- 对比第三方榜单（晨星十年期、银河证券长期奖项、金牛奖）做交叉验证
- 写出最终推荐报告（投资视角）：适合不同风险偏好投资者的 Top 5 名单

## 5. 迭代节奏（loop 怎么转）

每次 loop 推进按以下顺序：

```
1. Read RESEARCH-ROADMAP.md（确认本阶段任务）
2. 复制 templates/iteration-log.md → process/iter-NNN-YYYY-MM-DD.md
3. 执行本轮任务（搜索 → 抓取 → 记录 → 更新 INDEX）
4. 在 log 中写"本轮关键发现 / 未决问题 / 下一步"
5. 如果结论有更新，同步 output/INDEX.md
6. loop 触发下一轮
```

**单次迭代的最小交付**：
- 一份 `process/iter-NNN-*.md` 记录
- 至少 1 条 `output/INDEX.md` 的更新（新增/修改/删除候选）
- 至少 1 个未决问题的明确表述

## 6. 已知的陷阱与原则

### 6.1 陷阱清单

| 陷阱 | 应对 |
|---|---|
| 幸存者偏差：只看仍在管的经理，忽略已离任的明星 | 在"历史参考"栏保留离任者但标注 |
| 风格押注伪 alpha：2019-2021 押白酒/新能源的人看起来是 alpha，其实是 beta | 必做子区间测试 + 晨星归因数据交叉验证 |
| 多源数据冲突：天天基金/晨星/好买数据不一致导致判断混乱 | **单源原则**：只用晨星一个数据源 |
| 经理变更：同一只产品可能换过经理 | 只看经理任职区间内数据 |
| 一拖多：经理名下多只产品表现差异大 | 取代表产品或规模加权，明确披露口径 |
| 公司更名/产品转型：历史数据有断点 | 标注转型节点，不强行拼接 |
| 规模爆炸稀释收益：业绩好的基金会吸引大量申购导致收益下降 | 关注规模增长曲线，标注规模拐点 |
| 工具失忆：loop 重新启动时可能不知道有哪些 API/页面模式可用 | 维护 [`data/SEARCH-GUIDE.md`](./data/SEARCH-GUIDE.md)，每次新增模式都加进去 |
| 抓取数据全堆内存：单只基金 10 年约 2500 条 × 50 只 = 12.5 万行 | 立即落盘 `data/raw/nav/<code>.csv`，不在 evaluate_script 返回值里累积 |

### 6.2 工作原则

1. **可证伪优先**：每个结论必须有数据/来源支撑，标注证据等级（强/中/弱）
2. **多源交叉**：任何关键事实至少 2 个独立来源
3. **承认未知**：不能量化或不能确证的事，写到"未决问题"里，不要编
4. **小步快跑**：每次迭代交付小块，loop 推进；不要试图一次完成
5. **引用一切**：所有数据/陈述都要有 URL 或文件路径引用
