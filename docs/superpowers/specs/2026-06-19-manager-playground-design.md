# Manager Playground 设计文档（iter-007 · 2026-06-19）

> **目标**：为 funds-research 项目创建一个本地 HTML playground，让研究员能直观对比和查看已抓取的基金经理数据。

---

## 1. 背景与目标

### 1.1 问题

funds-research 项目已经积累了 3 份基金经理 JSON（郑希、武阳、刘元海，iter-006 v1.5 落盘），后续 iter-007+ 会持续扩张。但目前查看数据只能：

- 用 `cat json | jq` 看结构化输出
- 跑 `node data/validate-manager.js` 看校验报告
- 直接 `Read` 单个 JSON 文件

这些方式都不利于**横向对比**——研究的核心诉求是"找穿越周期的 alpha 候选"，对比是主要动作。

### 1.2 目标

提供一个**本地 HTML playground**，让研究员能：

1. **一眼对比**：3+ 位经理的关键指标并列展示
2. **深入查看**：点击某位经理后看到完整 13 个 schema 字段
3. **自动同步**：iter-007+ 新增的 JSON 自动出现在 playground 中（无需手动加白名单）

### 1.3 非目标（v1 不做）

- ❌ 编辑 JSON（v1 只读）
- ❌ 数据筛选（v1.1 才加）
- ❌ 多经理 JSON diff（v1.1 才加）
- ❌ 持仓表点击排序（v1.1 才加）
- ❌ 历史快照对比（v2.0+ 才考虑）

## 2. 架构总览

```
┌─────────────────────────────────────────────────────────────┐
│  Browser (chrome / firefox)                                  │
│  ┌─────────────────────┐  ┌─────────────────────────────┐   │
│  │  对比表（默认视图）  │  │  详情区（点击行后滚动到这里） │   │
│  └─────────────────────┘  └─────────────────────────────┘   │
│           ▲                          ▲                       │
│           └─────── fetch /api/managers ─────────┘            │
└──────────────────────────────┬──────────────────────────────┘
                               │ HTTP
┌──────────────────────────────┴──────────────────────────────┐
│  Node.js Server (server.js, ~80 行)                          │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  启动时：扫描 ../data/raw/morningstar/manager-*.json  │  │
│  │  缓存到内存（in-memory cache）                         │  │
│  └───────────────────────────────────────────────────────┘  │
│  Routes:                                                    │
│    GET /            → public/index.html                     │
│    GET /style.css   → public/style.css                      │
│    GET /app.js      → public/app.js                         │
│    GET /api/managers → 所有经理的合并 JSON                   │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 文件结构

```
playground/                           ← 项目根（与 data/ 同级）
├── server.js                         ← Node http 服务，~80 行
├── public/                           ← 静态资源
│   ├── index.html                    ← 主页面（含所有 section anchor）
│   ├── style.css                     ← 浅色主题 + chip + 表格样式
│   └── app.js                        ← 前端逻辑（fetch + render）
├── package.json                      ← "type": "module", 1 个 scripts
└── README.md                         ← 使用文档（如何启动）
```

### 2.2 数据流

1. **启动**：用户在 `playground/` 目录运行 `npm start`（或 `node server.js`）
2. **扫描**：server.js 启动时同步扫描 `../data/raw/morningstar/manager-*.json`，读取每个 JSON 的 `_meta`（name/managerId）建立列表
3. **首次请求**：浏览器访问 `http://localhost:8765/`，返回 `index.html`
4. **API 调用**：前端 `app.js` 启动后 `fetch('/api/managers')`，server 返回合并 JSON（包含所有经理的完整数据）
5. **渲染对比表**：`renderCompareTable(managers)` 填充 `<table>`
6. **点击行**：`onclick` 事件触发 `renderDetail(manager)` + `scrollIntoView`
7. **详情渲染**：按 schema 顺序填充 7 个 `<section>`（basic / 业绩 / 标签 / 行业 / 风格 / 持仓 / 基金）

## 3. API 设计

### 3.1 `GET /api/managers`

**响应**：JSON 数组，每个元素是单个经理的完整数据（与 `data/raw/morningstar/manager-<id>-<name>.json` 同结构）

**示例**：
```json
[
  { "_meta": {...}, "basic": {...}, ... },  // 郑希
  { "_meta": {...}, "basic": {...}, ... },  // 武阳
  { "_meta": {...}, "basic": {...}, ... }   // 刘元海
]
```

**排序**：按 `basic.annualReturnEquity` 降序（年化收益最高的排前面，最契合"找 alpha"主线）

**错误处理**：
- 扫描目录不存在 → 返回 `[]` + HTTP 200，前端显示空状态
- 单个 JSON 解析失败 → 跳过该文件 + 记录到 server 日志，前端显示成功加载的经理 + "X 个加载失败" 警告条

### 3.2 静态资源

- `GET /` → `public/index.html`
- `GET /style.css` → `public/style.css`
- `GET /app.js` → `public/app.js`

不依赖任何 CDN，所有 CSS/JS 都在本地。

### 4.2 交互流程

1. **页面加载** → fetch /api/managers → 渲染对比表
2. **鼠标悬停行** → 行背景变浅灰 + cursor: pointer
3. **点击行** → 
   - 该行高亮（蓝色边框 1 秒后淡出）
   - 详情区填充数据
   - `window.scrollTo({ top: detailEl.offsetTop, behavior: 'smooth' })`
4. **点击同一行再次** → 不做任何事（已展示）
5. **点击不同行** → 详情区内容切换 + 滚动

### 4.3 详情区 section 顺序

按"先摘要后细节"的认知负荷顺序，共 9 个 section：

1. **基本信息**（basic）：公司 / 学历 / bio / 投资年限 / AUM / 基金数 / 资产类型 / 管理类型 / 年化收益
2. **业绩标签**（labels.performance）：正/负/中分组展示
3. **风险回报**（riskReturn.current）：1Y 的 6 个指标卡片
4. **历年回报**（annualReturns）：10 年表格 + 基准 + 超额
5. **行业配置**（industryAllocation.current）：5-10 行业条形图（用 CSS div 宽度表示占比）
6. **风格箱**（styleBox.cells）：3x3 表格 + sizeBias/styleBias 标记
7. **持仓**（topHoldings.quarterly）：10 行表格（代码 / 名称 / 权重 / 首次买入 / 市值 / 份额变动 / 行业）
8. **重仓股持有期**（holdingPeriods.quarterly）：10 行表格
9. **管理基金列表**（funds）：所有产品（含代表产品高亮）

## 5. 错误处理

| 场景 | 表现 | 应对 |
|---|---|---|
| `data/raw/morningstar/` 目录不存在 | 对比表显示空状态："还没抓任何经理，先跑 `node data/parse-manager.js`" | 文案 + 命令提示 |
| 单个 JSON 解析失败 | 该经理跳过，其他照常显示 | 顶部黄色警告条："N 个 JSON 加载失败" |
| `/api/managers` 请求失败 | 对比表显示 "fetch 失败，请检查 server 是否运行在 :8765" | 文案 + 检查命令 |
| 数据字段缺失（某经理无 riskReturn 等） | 该单元格显示 "—" | null 检查 |
| 数字过大（如 +1635.66%） | 完整显示 | 不缩写 |
| 数字过小（如 0.36 亿） | 完整显示 | 不缩写 |

## 9. 风险与未决问题

| 风险 | 概率 | 应对 |
|---|---|---|
| iter-007+ 新增 JSON 后 server 没重启 | 中 | README 强调"添加新经理需重启 server"；未来 v1.5 加 watch 模式 |
| 静态资源路径在不同 OS 不一致 | 低 | 使用相对路径（`./style.css`） |
| 浏览器 fetch 跨域问题 | 0 | server 显式返回 `Access-Control-Allow-Origin: *`（同源则无所谓，但加上无害）|
| JSON 文件损坏 | 低 | 单文件错误不影响其他；顶部警告条 |
| server.js 端口 8765 被占用 | 中 | 启动时检测端口占用并提示 |
